# GAS ウェブアプリ - スライドプロンプト生成ツール

Google Apps Script（GAS）で作成したスライド画像生成プロンプトツール。スピーチ原稿から自動的にスライド構成を分析し、画像生成プロンプトを作成。さらに、Gemini 3 Pro Image (Nano Banana Pro) APIを使って直接画像を生成できます。

## 主な機能

### プロンプト生成
- **スピーチ原稿の自動分析**: Gemini 3 Pro APIで原稿を自動分析し、最適なスライド構成を提案
- **プロンプト自動生成**: 各スライドの画像生成プロンプトを自動作成
- **プロンプト直接編集**: 生成後もテキストエリアで自由に編集可能（自動保存）
- **自動英訳**: プロンプトを自動的に英語に翻訳（文脈を考慮した高品質翻訳）

### 画像生成
- **AI画像生成**: Gemini 3 Pro Image (Nano Banana Pro) APIで高品質画像を生成（約20円/枚）
- **サムネイル表示**: 生成画像をカード内にサムネイル表示
- **拡大モーダル**: サムネイルクリックで全画面表示
- **ダウンロード機能**: 生成画像をPNG形式でダウンロード可能

### カスタマイズ & 設定
- **85種類のデザインスタイル**: アート、アニメ、デザイン、歴史、SF、写真、ビジネス、自然、ムード、エンターテイメント、カスタム
- **自動カテゴリー展開**: 選択済みスタイルのカテゴリーを起動時に自動展開
- **テキスト表示設定**: 日本語/英語タイトル・メッセージの位置、配置、サイズ、色を調整
- **LocalStorage保存**: 全設定をブラウザーに自動保存（次回起動時に復元）
- **Markdownエクスポート**: `production_guide.md` 形式でダウンロード可能

## 前提条件

- Googleアカウント
- **Gemini API キー（有料プラン必須）**
  - ⚠️ **重要**: このツールは **Cloud Billing が有効な有料 API キー** が必要です
  - 無料の AI Studio キーでは `gemini-3-pro` シリーズを利用できません
  - [Google Cloud Console](https://console.cloud.google.com/) で請求先アカウントを設定してください
  - 使用モデル:
    - テキスト生成: `gemini-3-pro-preview` (スライド分析・翻訳)
    - 画像生成: `gemini-3-pro-image-preview` (Nano Banana Pro、約20円/枚)

## デプロイ手順

### 方法1: clasp（推奨）

```bash
# claspインストール（初回のみ）
npm install -g @google/clasp

# ログイン（初回のみ）
clasp login

# プロジェクト作成
clasp create --type webapp --title "スライドプロンプト生成ツール"

# ファイルをプッシュ
clasp push

# ウェブアプリとしてデプロイ
clasp deploy --description "v1.2"
```

### 方法2: 手動アップロード

#### 1. Google Apps Script プロジェクトを作成

1. [Google Apps Script](https://script.google.com/) にアクセス
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「スライドプロンプト生成ツール」に変更

#### 2. ファイルをアップロード

**Code.gs**
1. 左側の「ファイル」から「Code.gs」を開く
2. `Code.gs` の内容をすべてコピーして貼り付け

**PromptTemplates.gs**
1. 左側の「ファイル」メニューから「+」ボタンをクリック → 「スクリプト」を選択
2. ファイル名を「PromptTemplates」にする
3. `PromptTemplates.gs` の内容をすべてコピーして貼り付け

**Index.html**
1. 左側の「ファイル」メニューから「+」ボタンをクリック → 「HTML」を選択
2. ファイル名を「Index」にする
3. `Index.html` の内容をすべてコピーして貼り付け

#### 3. デプロイ

1. 右上の「デプロイ」→「新しいデプロイ」をクリック
2. 「種類の選択」で「ウェブアプリ」を選択
3. 設定:
   - **説明**: 「スライドプロンプト生成ツール v1.2」
   - **次のユーザーとして実行**: 「自分」
   - **アクセスできるユーザー**: 「全員」または「自分のみ」
4. 「デプロイ」をクリック
5. 表示されるURLをコピー（これがウェブアプリのURL）

## 使い方

### 基本フロー

1. **デプロイしたURLにアクセス**
2. **APIキー設定**（初回のみ）
   - 「🔑 Gemini APIキー設定」を展開
   - [Google AI Studio](https://aistudio.google.com/app/apikey)で取得したAPIキーを入力
   - 入力したキーはブラウザーに保存され、次回起動時に自動入力されます
3. **スピーチ原稿**にテキストを貼り付け（最大10,000文字）
4. **デザインテイスト選択**（85種類から複数選択可）
   - 選択済みカテゴリーは次回起動時に自動展開されます
5. **「プロンプトを生成」をクリック**
6. 生成されたプロンプトを確認・編集（textareaで直接編集可能）
7. **各スライドで画像生成**（オプション）
   - 「🎨 画像生成 [約20円/枚]」ボタンをクリック
   - サムネイルが表示されたらクリックして拡大
   - 「📥 ダウンロード」で画像を保存

### 料金について

- **プロンプト生成**: 従量課金（有料APIキー必須）
- **画像生成**: 約20円/枚（Gemini 3 Pro Image - Nano Banana Pro）
  - 16:9 アスペクト比
  - safetyFilterLevel: block_none
- ⚠️ **注意**: 無料のAI Studioキーでは利用できません

### 設定の保存

- すべての設定は**ブラウザーのLocalStorageに自動保存**されます
- 保存される項目:
  - Gemini APIキー
  - 85種類のデザインスタイル選択状態
  - テキスト表示オプション
  - テキスト設定（位置、配置、サイズ、色）
  - エフェクト設定
- 保存タイミング:
  - チェックボックス/セレクト変更時: **即座に保存**
  - テキスト入力: **500ms後に自動保存**
  - プロンプト生成ボタン押下時: **即座に保存**

## トラブルシューティング

### APIキーエラー
- APIキーが入力されているか確認（「🔑 Gemini APIキー設定」を展開）
- APIキーステータスが「✓ 設定済み」になっているか確認
- APIキーに十分なクォータがあるか確認

### エラー: "Failed to parse Gemini response"
- スピーチ原稿が入力されているか確認
- APIキーのクォータが残っているか確認

### 画像生成エラー
- **NO_API_KEY**: APIキーが未設定。基本設定でキーを入力
- **API利用制限**: レート制限（429エラー）の場合はしばらく待つ
- **無効なプロンプト**: 400エラーの場合はプロンプトを修正
- **料金**: 画像生成は約20円/枚かかります

### 生成が遅い
- プロンプト生成: Gemini APIは通常5〜10秒
- 画像生成: 10〜30秒（画像サイズと複雑さによる）
- ネットワーク状況を確認してください

### 日本語が文字化けする
- GASプロジェクトのタイムゾーンを「東京」に設定
- `Code.gs` の文字コードがUTF-8になっているか確認

### 設定が保存されない
- ブラウザーのLocalStorageが有効か確認
- プライベートブラウジングモードでは保存されません
- ブラウザーのキャッシュをクリアすると設定も消えます

## 技術スタック

- **バックエンド**: Google Apps Script (JavaScript, V8 Runtime)
- **フロントエンド**: HTML5 + CSS3 + Vanilla JavaScript
- **AI API**:
  - Gemini 3 Pro Preview (テキスト生成・翻訳)
  - Gemini 3 Pro Image / Nano Banana Pro (画像生成)
- **ストレージ**: LocalStorage (ブラウザー内保存)

## セキュリティ

- **APIキー**: ユーザーがフロントエンドで入力（サーバーにハードコーディングしない）
- **LocalStorage保存**: APIキーはブラウザーのLocalStorageに保存
- **パスワード型入力**: `type="password"` で入力内容を隠す
- 画像生成時のsafety filter: `block_none`（プレゼンテーション用途想定）
- ウェブアプリのアクセス権限は設定で制御可能
